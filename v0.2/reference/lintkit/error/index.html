    Error

    lintkit custom errors.

    These errors should not be caught, but rather used to inform:

    • linter developers about common mistakes they made during rules creation (most common)
    • linter users about common mistakes they made (e.g. incorrect noqa ignores usage)

    For the first example we might have:

    import ast
import lintkit


class MyRule(
    lintkit.check.Regex,
    lintkit.loader.Python,
    lintkit.rule.Node,
    code="123",  # offending line (should be an integer)
):
    def regex(self):
        return ".*"  # match everything

    def values(self):
        nodes = self.getitem("nodes_map")[ast.ClassDef]
        for node in nodes:
            yield lintkit.Value.from_python(node.name, node)

    which raises:

    lintkit.error.CodeNotIntegerError:
    Rule 'MyRule' has code '123' which is of type 'str',
    but should be a positive `integer` .

    while the second example might be (file being linted):

    def bar():
    pass


# noqa-start: MYRULE10
def foo():
    pass


# No noqa-end specified

    which raises:

    lintkit.error.IgnoreRangeError:
    End of ignore range missing, please specify it.
    Start of the range was at line `4` with content: `# noqa-start: MYRULE10`.

    lintkit.error.NotSubclassError ¶

    NotSubclassError()

    Bases: LintkitError

    Raised when the registered rule is not an appropriate subclass.

    Info

    Created rule (via code=<NUMBER>) has to inherit from lintkit.rule.Rule __AND [lintkit.loader.Loader].

    Warning

    Python's typing does not support intersection types, hence this is checked dynamically.

    Tip

    This error is raised automatically by lintkit, no need for explicit raising.

    The following would be an offending call:

    import lintkit


# Does not inherit from `loader.Loader`
class MyRule(lintkit.rule.Node, code=-1):
    pass

    lintkit.error.IgnoreRangeError ¶

    IgnoreRangeError(file, start, line)

    Bases: LintkitError

    Raised when the end of the ignore range is missing.

    Note

    Informs the user when the noqa-range/range ignore was started in a file, but was not explicitly ended.

    Tip

    This error is raised automatically by lintkit when rules are ran, no need for explicit raising.

    For this lintkit.settings.ignore_span_start and lintkit.settings.ignore_span_end:

    import lintkit

# Anything between igstart: <NAME><CODE> and igend: <NAME><CODE>
lintkit.settings.ignore_span_start: str = ".* igstart: .*{name}{code}.*"
lintkit.settings.ignore_span_end: str = ".* igend: .*{name}{code}.*"

    The following spans would throw IgnoreRangeError:

    # igstart: BU137


def foo():
    pass


# Different error code!
# igend: BU213


# igstart: BU137
def bar():
    pass


# No igend at all!

    This would throw:

    End of ignore range missing, please specify it.
Start of the range was at line '0' with content: '# igstart: BU137'.
    Note

    See settings.ignore_span_start and settings.ignore_span_end for more information.
